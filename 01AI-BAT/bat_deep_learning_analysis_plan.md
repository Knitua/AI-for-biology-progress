# 蝙蝠基因组深度学习分析计划

日期：2026-05-16

## 1. 项目背景

合作方提供了若干代表性蝙蝠基因组测序文章、Bat1K 相关入口以及当前已有的蝙蝠基因组数据，希望评估是否可以开展深度学习分析。

初步判断：该方向可行，但不建议第一阶段直接从全量原始测序 reads 开始处理。更合适的切入点是利用已经组装和注释好的蝙蝠基因组数据，结合预训练基因组或蛋白基础模型，围绕明确的生物学问题开展候选基因筛选和序列表征分析。

推荐第一阶段主题：

> 基于预训练基因组/蛋白基础模型，对蝙蝠免疫和病毒耐受相关基因进行深度学习表征与候选基因筛选。

该方向与现有蝙蝠基因组研究重点吻合，包括病毒耐受、天然免疫、炎症调控、DNA 损伤修复、长寿和适应性进化等。

- `Bat Species Database  Sequencing Status.xlsx`，记录蝙蝠物种、采集、测序、组装和注释状态。

Bat1K 相关入口：

## 3. 推荐分析目标

第一阶段建议聚焦于：

1. 建立代表性蝙蝠物种和非蝙蝠哺乳动物对照数据集。
2. 整理免疫和病毒耐受相关候选基因集合。
3. 使用预训练 DNA 或蛋白基础模型提取序列 embedding。
4. 比较蝙蝠与其他哺乳动物在候选基因上的序列表示差异。
5. 筛选可能与病毒耐受、炎症调控或免疫适应相关的候选基因。

重点关注的生物学方向：

- innate immunity
- interferon pathway
- interferon-stimulated genes, ISGs
- viral entry and restriction factors
- inflammatory response
- DNA damage repair
- antiviral response
- immune tolerance
- longevity-related immune regulation

可优先关注的代表性基因或通路包括：`IFN`、`ISG15`、`OAS`、`MX`、`TLR`、`RIG-I/DDX58`、`MDA5/IFIH1`、`NLRP3`、`STING/TMEM173`、`cGAS/MB21D1`、`APOBEC`、`TRIM`、DNA repair genes 等。正式分析时应根据注释质量和 ortholog 映射结果确定最终列表。

## 4. 技术路线

### 4.1 数据准备

优先下载和整理以下数据类型：

- genome FASTA：基因组序列。
- GFF/GTF：基因结构注释。
- CDS FASTA：编码序列。
- protein FASTA：蛋白序列。
- gene ID mapping：不同数据库之间的基因 ID 对照。
- ortholog table：蝙蝠和其他哺乳动物的一对一同源基因关系。

第一阶段不建议下载全量原始 FASTQ/SRA。只有在后续需要重新组装、重新注释或验证特定样本质量时，再考虑下载原始 reads。

### 4.2 物种选择

第一阶段建议选择 10-20 个物种：

- 6-12 个代表性蝙蝠物种。
- 3-8 个非蝙蝠哺乳动物对照物种。

蝙蝠物种应优先满足：

- 已有较高质量组装。
- 已有基因注释。
- 能覆盖主要蝙蝠类群。
- 与病毒耐受、免疫或相关论文研究背景有关。

非蝙蝠对照可考虑：

- human
- mouse
- rat
- dog
- cow
- pig
- horse
- shrew

具体物种以可获得的注释质量和 ortholog 完整性为准。

### 4.3 候选基因集合构建

候选基因来源：

- 已发表蝙蝠基因组论文中的免疫、病毒耐受和正选择候选基因。
- GO、KEGG、Reactome 中与免疫、病毒响应、炎症和 DNA 损伤修复相关的基因集。
- 人类和小鼠中已知抗病毒或炎症调控基因。
- 多物种 ortholog 映射后可可靠比较的一对一同源基因。

输出：

- 候选基因清单。
- 每个物种对应的 ortholog 序列。
- 每个候选基因的 CDS 和 protein 序列。

### 4.4 深度学习模型选择

推荐优先使用成熟预训练模型做 embedding，而不是从零训练大模型。

| 模型 | 类型 | 推荐用途 | 资源需求 |
|---|---|---|---|
| DNABERT-2 | DNA 模型 | CDS、启动子、调控区 embedding | 8-24 GB GPU 可试点 |
| Nucleotide Transformer | DNA 模型 | DNA 序列功能表征 | 24-80 GB GPU，视模型大小而定 |
| ESM2 650M | 蛋白模型 | 蛋白序列 embedding，推荐第一阶段使用 | 8-24 GB GPU 可试点 |
| ESM2 3B | 蛋白模型 | 更强蛋白表征 | 24-48 GB GPU 更稳 |
| Evo / Evo 2 | 长序列基因组模型 | 前沿长上下文 DNA/RNA/蛋白建模 | 80 GB 或多卡/云端更现实 |

第一阶段推荐组合：

- 蛋白主线：ESM2 650M。
- DNA/CDS 辅助主线：DNABERT-2 或较小版本 Nucleotide Transformer。

这样可以在单张 24 GB 显存 GPU 上完成可行性试点。

### 4.5 分析方法

建议分析流程：

1. 从注释文件中提取候选基因对应的 CDS 和 protein 序列。
2. 对每条候选蛋白序列使用 ESM2 生成 embedding。
3. 对候选 CDS 或调控序列使用 DNABERT-2/Nucleotide Transformer 生成 embedding。
4. 按基因和物种汇总 embedding。
5. 比较蝙蝠和非蝙蝠哺乳动物的 embedding 差异。
6. 进行聚类、降维可视化、分类建模或异常检测。
7. 结合功能注释、通路富集、正选择结果和论文证据，筛选优先候选基因。

可使用的下游方法：

- PCA、UMAP、t-SNE 可视化。
- hierarchical clustering。
- random forest、XGBoost、logistic regression 或轻量 MLP 分类。
- one-class/anomaly detection 用于寻找蝙蝠特异偏离序列。
- enrichment analysis 用于解释候选基因功能。

## 5. 数据量与资源需求

### 5.1 小规模试点

范围：10-20 个物种，包括若干蝙蝠和非蝙蝠哺乳动物对照。

| 数据类型 | 估计规模 |
|---|---:|
| genome FASTA | 20-50 GB |
| GFF/GTF | 1-5 GB |
| CDS FASTA | 1-3 GB |
| protein FASTA | < 1 GB |
| 候选基因子集 | 几 MB 到几十 MB |
| embedding 输出 | 1-10 GB |
| 推荐工作空间 | 50-200 GB |

如果只做蛋白序列和候选基因分析，10-50 GB 工作空间即可启动。

### 5.2 中等规模分析

范围：本地 Excel 中 91 个已完成测序、组装和注释的物种。

| 数据类型 | 估计规模 |
|---|---:|
| genome FASTA | 150-250 GB |
| GFF/GTF | 5-30 GB |
| CDS/protein FASTA | 5-15 GB |
| embedding 输出 | 10-100 GB |
| 索引、缓存、中间文件 | 100 GB 以上 |
| 推荐工作空间 | 500 GB-1 TB |

### 5.3 原始 SRA 全量处理

范围：`PRJNA489245` 下当前可检索的 74 条 SRA run。

| 项目 | 估计规模 |
|---|---:|
| SRA 归档数据 | 约 1.73 TB |
| 解压 FASTQ 后 | 可能 4-8 TB |
| 组装和中间文件 | 可能 10 TB 以上 |
| 推荐内存 | 128-512 GB 以上 |
| 推荐 CPU | 数十核心 |

该路线适合重新组装或重新注释，不适合作为第一阶段深度学习试点。

## 6. 第一阶段试点方案

建议第一阶段周期目标：

1. 筛选 10-20 个代表物种。
2. 获取对应 protein FASTA、CDS FASTA 和注释文件。
3. 建立免疫和病毒耐受候选基因清单。
4. 完成 ortholog 映射和候选序列提取。
5. 使用 ESM2 650M 和 DNABERT-2/Nucleotide Transformer 生成 embedding。
6. 做蝙蝠 vs 非蝙蝠对照的可视化和初步分类。
7. 输出优先候选基因和可解释的生物学结果。

第一阶段最小可交付结果：

- 物种和数据清单。
- 候选基因清单。
- 候选蛋白/CDS 序列文件。
- embedding 矩阵。
- PCA/UMAP 聚类图。
- 蝙蝠特异候选基因排序表。
- 简短结果报告。

## 7. 预期结果

预期能够回答的问题：

- 预训练模型的序列表示是否能区分蝙蝠和非蝙蝠哺乳动物的免疫相关基因。
- 哪些免疫或病毒耐受相关基因在蝙蝠中呈现明显序列表征偏离。
- 哪些基因适合进一步做正选择、结构建模或实验验证。
- 当前公开组装和注释数据是否足以支撑更大规模 Bat1K 深度学习分析。

可能形成的结果形式：

- 候选基因优先级表。
- 基因/物种 embedding 可视化。
- 蝙蝠特异免疫相关基因集合。
- 与已发表论文结果的交叉验证。
- 后续扩大到 91 个已完成物种的分析建议。

## 8. 风险与注意事项

主要风险：

- 不同物种注释质量不一致，可能影响候选基因提取。
- ortholog 映射错误会影响跨物种比较。
- 深度学习 embedding 有表征能力，但需要结合生物学证据解释。
- 如果没有明确生物学问题，仅做全基因组 embedding 容易产出难解释结果。
- 大模型推理成本较高，应先用小模型和候选基因子集验证可行性。

控制策略：

- 第一阶段只选高质量组装和注释物种。
- 优先使用一对一 ortholog。
- 同时保留传统比较基因组证据作为解释支撑。
- 先做蛋白模型，再扩展到 DNA/CDS/调控区。
- 不在第一阶段下载全量 SRA。
