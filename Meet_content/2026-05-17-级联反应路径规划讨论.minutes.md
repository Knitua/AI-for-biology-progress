## 一句话概括

本次讨论围绕“级联/多步反应路径规划中，模型应如何参与控制、筛选与修饰”展开。核心判断是：不要让模型从零逐步盲搜整条级联路线，更可行的路线是先由已有 retrosynthesis/auto planner 给出候选路径，再由模型做全局级联可行性评估、步骤合并、条件统一和路径修饰。

## 核心结论

1. 单步逐步搜索不适合级联反应规划。
   如果每一步都独立生成多个候选反应类型，很容易出现第三步 output 无法作为第四步 input、上下步条件不兼容等问题。对项目推进而言，不能把任务简单拆成“每一步预测一个反应”，而要把整条路径作为对象评估。

2. 级联规划必须具有全局路径视角。
   会议中反复强调，模型需要在反应开始时就看到整条路线大概有多少步、哪些步骤可能合并、哪些条件需要统一。否则模型只能局部判断当前 action 是否合理，无法判断其对后续步骤的影响。

3. 当前数据规模不足以支撑直接训练一个强级联规划模型。
   只依赖现有几千条级联数据去训练或微调，容易得到不可靠的“骨架”或路线。后续应避免把主线押在从小数据中直接学出完整规划能力上。

4. 小模型微调和蒸馏路线需要谨慎。
   讨论中提到 LLaMA 3.1、小模型控制、on-policy distillation 等方向，但大家对“小模型能否复制顶级大模型输出”和“微调用什么任务才有意义”持保留态度。短期更现实的是借助强 API 或现成模型做评估与选择。

5. 更可行的方案是“已有路线评估 + 级联修饰”。
   让已有 planner 或老师组模型先生成 top 100 路线，再判断哪些路线可改造成级联，哪些步骤可缩短，哪些条件可通过更换催化剂、酶或反应条件进行合并。这比从零生成级联路线更稳，也更容易验证。

6. 不建议强行向 auto planner 注入高权重人工候选。
   如果不给人工候选很高权重，planner 可能选不到；如果强行给高权重，又可能破坏原有规划逻辑。因此这条路更像临时补丁，不适合作为主线方法。

## 已形成的共识

- 级联反应规划的核心难点不是单步反应预测，而是跨步骤兼容性、条件统一和全局路线结构。
- 单步模型或传统逆合成模型给出的候选，天然不保证能构成可执行的级联路线。
- 当前更应该做“路线是否可级联”的评估器，而不是直接训练一个从零生成级联路线的模型。
- LLM 可以作为选择器、评估器或修饰器参与流程，但需要给它明确的候选路线、上下文和判断标准。
- 需要用 cascade/test set 一类数据先做验证，避免继续停留在概念讨论。

## 仍在讨论的问题

- 如何定义“级联可行性”的评价函数仍不清楚。尤其是当前 action 是否好，往往取决于后续路径，难以只根据当前状态判断。
- LLM 应该做候选选择、路线重排、条件合并，还是只做解释和打分，仍需进一步界定。
- 如果使用小模型或蒸馏，训练任务如何构造、目标输出是什么、是否能稳定复现强模型判断，还没有结论。
- 现有 planner 输出的 top 100 路线是否包含足够多可改造为级联的路线，需要用实际数据验证。
- 公开版本中 LLaMA 3.1 或类似模型到底在已有工作里起什么作用，仍需查清楚。

## 可行方向

1. 候选路线级联可行性评估器。
   输入是已有 planner 给出的多条候选合成路线；模型判断每条路线是否可能改造成级联路线，输出可级联概率、关键限制步骤和不兼容原因。验证方式是用人工标注或 cascade test set 检查排序与判断准确性。

2. 路线缩短与节点合并。
   输入是一条多步路线及每步反应类型、底物/产物、条件；模型识别可合并的相邻步骤或节点，输出缩短后的路线和修改理由。验证方式是检查合并后上下步物质连接、条件兼容性和化学合理性。

3. 条件统一与规避建议。
   输入是路线中各步温度、催化剂、酶、溶剂等条件；模型判断哪些步骤条件冲突、哪些条件可替换或统一，输出建议的条件调整方案。验证方式是由有机/催化背景成员复核。

4. 强模型辅助的候选选择。
   输入为每一步或每条路线的候选集合；LLM 根据全局路径上下文选择较优候选，而不是只看局部反应。验证方式是比较有无全局上下文时的路线连贯性和可执行性。

## 暂不推荐的方向

- 从零训练完整级联路线生成模型：数据量不足，且评价函数难定义。
- 每一步独立搜索并拼接成整条路线：上下步输入输出和条件很容易不兼容。
- 强行给 auto planner 注入人工高权重候选：容易破坏 planner 原有分布和选择逻辑。
- 直接做强化学习 policy：当前难以定义级联语境下的即时 reward，单步 action 好坏依赖后续路径。
- 小模型蒸馏作为短期主线：蒸馏效果不确定，小模型未必能复制强模型的复杂判断。

## 总结

本次讨论聚焦于级联/多步反应路径规划中模型的角色定位。大家基本认为，级联规划不能简单拆成逐步反应预测，因为每一步的产物、条件和后续步骤高度耦合；如果模型只在局部给出候选，容易得到上下步不兼容、条件无法统一的路线。

更稳妥的推进方式是先借助已有 retrosynthesis 或 auto planner 产生候选路线，再将研究重点放在路线级评估与修饰上：判断哪些路线可以改造成级联，哪些步骤可以合并或缩短，哪些条件可以通过更换催化剂、酶或反应条件进行统一。这样可以绕开当前级联数据量不足、从零训练模型不稳定的问题。

下一步建议先构建一批 cascade/test set 路线，明确路线字段和人工标注标准，再用强模型或 LLM prompt 做小规模评估验证。若路线级判断效果稳定，再考虑是否进一步做微调、蒸馏或专门的选择器模型。
