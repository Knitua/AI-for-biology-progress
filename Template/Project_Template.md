# 项目详情页模板

> 使用方式：复制本模板到项目文件夹的 `README.md`，替换所有 `待填写` 内容。正式项目页应直接展示项目本身，不要写“当前已有某文件”“根据某 PDF”“材料显示”等内部整理语言。

## 基本信息

| 字段 | 内容 |
|---|---|
| 项目方向 | 待填写：用 1 句话概括研究方向，例如“抗冻肽候选发现与实验验证” |
| 当前状态 | 待填写：如“方案设计中 / 数据资源成型 / 结果推进中 / 验证阶段 / 可展示成果” |
| 负责人/参与者 | 待填写 |
| 更新时间 | 待填写：YYYY-MM-DD |

## 研究问题

待填写：用 1-3 段说明项目要回答的科学问题。

写法建议：

- 先说明研究对象是什么。
- 再说明关键变量、关键机制或关键预测目标。
- 最后说明当前阶段最需要解决的判断问题。

避免写法：

- 不要写“我阅读了某材料后发现”。
- 不要写“这个文件里记录了”。
- 不要把项目页写成资料清单。

## 研究意义

待填写：说明这个项目为什么值得做。

可从以下角度选择 2-3 个展开：

- 生物机制理解价值。
- 疾病预测或临床转化价值。
- 药物发现、递送、蛋白设计或功能肽发现价值。
- 数据资源、方法平台或团队能力建设价值。
- 是否适合作为团队对外展示或后续招新项目。

## 项目目标

- 待填写：短期目标，建议是 2 周到 1 个月内可推进的目标。
- 待填写：中期目标，建议是阶段性路线图、候选清单、模型流程或实验验证方案。
- 待填写：可展示产出，例如结果表、流程图、候选列表、benchmark、实验结果或项目图谱。

## 技术路线

1. 待填写：数据、样本、文献、实验记录或计算资源如何进入项目。
2. 待填写：如何把原始信息转化为结构化问题、字段、标签、候选集或任务。
3. 待填写：核心分析、模型、实验、验证或评估流程。
4. 待填写：如何形成阶段性结果和可展示图表。
5. 待填写：后续如何扩大规模、验证机制或转入实验/应用。

## 当前进展

待填写：直接总结团队已经完成的工作、已经得到的结果、已经形成的路线和已经发现的问题。

推荐写成：

- 已完成什么数据/样本/候选/模型/实验/分析。
- 已得到哪些数字化结果或可展示结论。
- 已形成哪些路线、框架或判断边界。
- 哪些结果可以向老师展示，哪些结果还需要谨慎表述。

避免写成：

- “当前已有某某 PDF/Word/Excel”。
- “某讨论稿里记录了”。
- “下一步需要阅读材料”。
- “这篇文章说明了”。

如果材料来自上传文件，也要把它转化成项目事实，例如：

- 不写：“当前已有 `xxx.pdf`，内容显示数据库包含 40599 条记录。”
- 应写：“数据库已整理 40599 条器件记录，并完成核心字段覆盖和物理范围验证。”
